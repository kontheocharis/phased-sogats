# Phased second-order generalised algebraic theories


A PSOGAT is like a SOGAT but we have access to a meet semilattice of open
modalities.

More precisely, the PSOGAT theory of signatures over a meet semilattice `Φ`
(called a *phase algebra*) has all the type formers of the SOGAT theory of
signatures, but additionally gives access to one open modality for each element
in `Φ`, such that the ordering of `Φ` determines the relations between the open
modalities.

This repository formalises this theory of signatures, and its translation to
SOGATs (WIP).

## Phase algebra

A phase algebra `Φ` is a meet semilattice, given by the following data:

```
Phase : Set
_∧_ : Phase → Phase → Phase
⊤ : Phase
```

along with associativity, commutativity, ⊤ neutrality, and idempotency for `_∧_`.

## PSOGAT ToS

In the SOGAT ToS we have three universes: `U` for declaring sorts, `Uᴿ` for
declaring representable sorts, and `Ty` which is the sort for signatures
themselves.

The PSOGAT ToS over a phase algebra `Φ` extends the SOGAT ToS with a (weak) meet
semilattice homomorphism `Φ → PROPᴿ`, and an open modality for each `P : Phase`:

```
In : Phase → Propᴿ
prod : (In a × In b) ≃ In (a ∧ b)
top : In ⊤

Πᴾ : (P : Phase) → (In P → Ty) → Ty
Πᵁᴾ : (P : Phase) → (In P → Tm U) → Tm U
Πᴿᴾ : (P : Phase) → (In P → Tm Uᴿ) → Tm Uᴿ
```

where each of `Πᴾ`, `Πᵁᴾ` and `Πᴿᴾ` come with the usual `lam`/`app` isomorphism.

To simplify things, when writing down a PSOGAT, we can use the notation `(@P x :
A) → B` to mean `(x : Πᴿᴾ P (λ _. A)) → B`, and we can omit the `lam`/`app`
coercions since these can always be inserted automatically.

## Example: erasure

Let's take the meet semilattice `E` generated by `{0, ω}` where `ω` is the top
element.

We can write down the definition of 'type theory with erasure' as presented
[here](https://arxiv.org/abs/2605.00655) as

```hs
@0 Ty : U
Tm : (@0 Ty) → Uᴿ

-- Pi with erased domain
@0 Π0 : (A : Ty) → (Tm A → Ty) → Ty
(lam, app) : ((@0 x : Tm A) → Tm (B x)) ≃ Tm (Π0 A B)

-- Pi with runtime domain
@0 Πω : (A : Ty) → (Tm A → Ty) → Ty
(lam, app) : ((x : Tm A) → Tm (B x)) ≃ Tm (Πω A B)
```

When we generate the SOGAT for this, we get (up to some prettifying and reshuffling):

```hs
-- The translation gives
--   In : Phase → Upropᴿ 
--   prod : (In a × In b) ≃ In (a ∧ b)
--   top : In ⊤
-- and here we have Phase := {0, ω} and ⊤ := ω
-- which reduces to In : {0} → Uᴿ since ⊤ always holds.
In0 : Upropᴿ

-- Each sort in the original PSOGAT directly gives a sort here.
Ty' : In0 → U
-- I will shorthand this:
Ty : U = (i : In0) → Ty' i

-- For each representable sort in the original PSOGAT, and each phase, we get 
-- a new representable sort, and we also get an iso that relates them via In.
Tmω : Ty → Uᴿ
Tm0 : Ty → Uᴿ
(↓, ↑) : ((i : In0) → Tmω A) ≃ Tm0 A

-- Each time we bind a representable sort with an open modality in the original
-- PSOGAT (e.g. @0 Tm A), this is translated to the generated representable
-- sort for *that* phase. For that example, Tm0 A.

-- Pi with erased domain
Π0 : (i : In0) → (A : Ty' i) → (Tmω A → Ty' i) → Ty' i
(lam, app) : ((x : Tm0 A) → Tmω (λ i . B i (↑ x i))) ≃ Tmω (λ i. Π0 i (A i) (B i))

-- Pi with runtime domain
Πω : (i : In0) → (A : Ty' i) → (Tmω A → Ty' i) → Ty' i
(lam, app) : ((x : Tmω A) → Tmω (λ i . B i x)) ≃ Tmω (λ i . Πω i (A i) (B i))
```

This is equivalent to the SOGAT given in the referenced paper, up to the
insertion of the ↑↓ isomorphism, and choice of `Ty` vs `Ty'`.

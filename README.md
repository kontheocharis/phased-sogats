# Phased second-order generalised algebraic theories


A PSOGAT is like a SOGAT but we have access to a meet semilattice of open
modalities.

More precisely, the PSOGAT theory of signatures over a meet semilattice `Œ¶`
(called a *phase algebra*) has all the type formers of the SOGAT theory of
signatures, but additionally gives access to one open modality for each element
in `Œ¶`, such that the ordering of `Œ¶` determines the relations between the open
modalities.

This repository formalises this theory of signatures, and its translation to
SOGATs (WIP).

## Phase algebra

A phase algebra `Œ¶` is a meet semilattice, given by the following data:

```
Phase : Set
_‚àß_ : Phase ‚Üí Phase ‚Üí Phase
‚ä§ : Phase
```

along with associativity, commutativity, ‚ä§ neutrality, and idempotency for `_‚àß_`.

## PSOGAT ToS

In the SOGAT ToS we have three universes: `U` for declaring sorts, `U·¥ø` for
declaring representable sorts, and `Ty` which is the sort for signatures
themselves.

The PSOGAT ToS over a phase algebra `Œ¶` extends the SOGAT ToS with a (weak) meet
semilattice homomorphism `Œ¶ ‚Üí PROP`, and an open modality for each `P : Phase`:

```
In : Phase ‚Üí Prop·¥ø
prod : (In a √ó In b) ‚âÉ In (a ‚àß b)
top : In ‚ä§

ŒÝ·¥æ : (P : Phase) ‚Üí (In P ‚Üí Ty) ‚Üí Ty
ŒÝ·µÅ·¥æ : (P : Phase) ‚Üí (In P ‚Üí Tm U) ‚Üí Tm U
ŒÝ·¥ø·¥æ : (P : Phase) ‚Üí (In P ‚Üí Tm U·¥ø) ‚Üí Tm U·¥ø
```

where each of `ŒÝ·¥æ`, `ŒÝ·µÅ·¥æ` and `ŒÝ·¥ø·¥æ` come with the usual `lam`/`app` isomorphism.

To simplify things, when writing down a PSOGAT, we can use the notation `(@P x :
A) ‚Üí B` to mean `(x : ŒÝ·¥ø·¥æ P (Œª _. A)) ‚Üí B`, and we can omit the `lam`/`app`
coercions since these can always be inserted automatically.

## Example: erasure

Let's take the meet semilattice `E` generated by `{0, œâ}` where `œâ` is the top
element.

We can write down the definition of 'type theory with erasure' as presented
[here](https://arxiv.org/abs/2605.00655) as

```hs
@0 Ty : U
Tm : (@0 Ty) ‚Üí U·¥ø

-- Pi with erased domain
ŒÝ0 : (A : Ty) ‚Üí ((@0 Tm A) ‚Üí Ty) ‚Üí Ty
(lam, app) : ((@0 x : Tm A) ‚Üí Tm (B x)) ‚âÉ Tm (ŒÝ0 A B)

-- Pi with runtime domain
ŒÝœâ : (A : Ty) ‚Üí (Tm A ‚Üí Ty) ‚Üí Ty
(lam, app) : ((x : Tm A) ‚Üí Tm (B x)) ‚âÉ Tm (ŒÝœâ A B)
```

When we generate the SOGAT for this, we get (up to some prettifying and reshuffling):

```hs
-- The translation gives
--   In : Phase ‚Üí Uprop·¥ø 
--   prod : (In a √ó In b) ‚âÉ In (a ‚àß b)
--   top : In ‚ä§
-- and here we have Phase := {0, œâ} and ‚ä§ := œâ
-- which reduces to In : {0} ‚Üí U·¥ø since ‚ä§ always holds.
In0 : Uprop·¥ø

-- Each sort in the original PSOGAT directly gives a sort here.
Ty' : In0 ‚Üí U
-- I will shorthand this:
Ty : U = (i : In0) ‚Üí Ty' i

-- For each representable sort in the original PSOGAT, and each phase, we get 
-- a new representable sort, and we also get an iso that relates them via In.
Tmœâ : Ty ‚Üí U·¥ø
Tm0 : Ty ‚Üí U·¥ø
(‚Üì, ‚Üë) : (In0 ‚Üí Tmœâ A) ‚âÉ Tm0 A

-- Each time we bind a representable sort with an open modality in the original
-- PSOGAT (e.g. @0 Tm A), this is translated to the generated representable
-- sort for *that* phase. For that example, Tm0 A.

-- Pi with erased domain
ŒÝ0 : (A : Ty) ‚Üí (Tm0 A ‚Üí Ty) ‚Üí Ty
(lam, app) : ((x : Tm0 A) ‚Üí Tmœâ (B x)) ‚âÉ Tmœâ (ŒÝ0 A B)

-- Pi with runtime domain
ŒÝœâ : (A : Ty) ‚Üí (Tmœâ A ‚Üí Ty) ‚Üí Ty
(lam, app) : ((x : Tmœâ A) ‚Üí Tmœâ (B x)) ‚âÉ Tmœâ (ŒÝœâ A B)
```

This is exactly the SOGAT given in the referenced paper.
